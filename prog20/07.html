Tarea: Arreglos ordenados o no

El objetivo de este ejercicio es escribir un programa no interactivo cuya salida siga los ejemplos que siguen:

[root@localhost ~]# ./arr
[2,2,0,7,2]
El arreglo no esta ordenado porque 2 > 0.

[root@localhost ~]# ./arr
[6,6,5,7,1]
El arreglo no esta ordenado porque 6 > 5.

[root@localhost ~]# ./arr
[0,7,3,2,7]
El arreglo no esta ordenado porque 7 > 3.

[root@localhost ~]# ./arr
[1,1,1,6,6]
El arreglo esta ordenado.

Es decir, el programa genera los valores del arreglo aleatoriamente.

1.1 Inicializar aleatoriamente un arreglo

En un programa arr.c escribí lo siguiente.

  1. Definí una constante T de valor 5.
  2. Declará un arreglo a de tamaño T
  3. Llená aleatoriamente el arreglo a con valores entre 0 y 9.
  4. Imprimí el arreglo tal como lo hace el programa dado en este enunciado (hacelo en un segundo bucle).

1.2 Comprobar si el arreglo está ordenado

Agregá a tu programa el código necesario para reproducir el comportamiento del programa dado.

Un arreglo a está ordenado si para todos par de elementos consecutivos (ai, ai + 1), tenemos ai ≤ ai + 1.

Para ello necesitás un bucle más en tu programa.

Necesitás una variable aparte para guardar el dato de si el arreglo está ordenado. Podés llamarla ord y inicializarla a 1 antes de recorrer el arreglo, y cambiarla a 0 si encontrás dos elementos consecutivos tales que ai > ai + 1. Si después de terminar la ejecución de ese bucle, ord sigue valiendo 1, entonces el arreglo está ordenado.

Después del bucle, imprimí un mensaje que indica si el arreglo está ordenado, y en el caso contrario, indicá el primer par de elementos consecutivos que no están ordenados.
