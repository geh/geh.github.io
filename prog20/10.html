Parte 1: Primeras funciones con arreglos

En un programa arreglos.c definir una constante MAX para el tamaño de todos nuestros arreglos:

#define MAX 10

Definir las funciones siguientes:

  • void mostrar(const int a[]): muestra el arreglo en una sola línea
  • void llenar_random(int a[]): una función que llena el arreglo de valores aleatorios entre 0 y 99.

Luego, en el main, declarar un arreglo, llenarlo con llenar_random y mostrarlo con mostrar.

Luego, definir una función void invertir(const int a[], int b[]), que no modifique la lista a y que escribe en la lista b el inverso de a.

Usarla desde la funcion main de la forma siguiente:

...
int x[MAX];
int y[MAX];
...
llenar_random(x);
mostrar(x);
invertir(x,y);
mostrar(y);
...

Parte 2: Posición de un elemento

En un programa busqueda.c, definir una constante ERROR con el valor -1.

Implementar la función int buscar(const int arr[], const int elem), que realiza la búsqueda de un elemento en un arreglo.

Esta función devuelve un valor de tipo int, que tiene que ser:

  • el primer i tal que elem == arr[i], si existe
  • el valor ERROR, cuando el elemento buscado no está en el arreglo.

Unos ejemplos:

arr elem buscar(arr,elem)
[2,10,4] 2 0
[2,10,4] 4 2
[3,7,7] 7 1
[3,7,3] 3 0
[3,7,3] 4 ERROR

Probar esta función desde el main sobre dos arreglos definidos manualmente:

#define ERROR -1
int main(){
   int arr[3] = {2,10,4};
   int elem = 2;
   int resultado = buscar(arr,elem);
   if (resultado == ERROR)
     printf("El elemento no está en el arreglo.\n");
   else
     printf("El elemento está, su posición es %d.\n", resultado);
   return 0;
}

Parte 3: Visualizar ordenamiento de arreglos

Hacer este ejercicio dentro de JSLinux, y con el editor de texto vi.

Copiar el código siguiente en un programa llamado ordenar.c.

#include <stdio.h>
#include <stdlib.h>
#include <time.h>
#define MAX 10
#define AZUL    "\e[104m"
#define MAGENTA "\e[105m"
#define RESET   "\e[0m"

/* Algoritmo de Fisher-Yates */
void mezclar(int a[]){
    int i,j,t;
    for (i=0; i<MAX; i = i+1) a[i] = i;
    for(i=MAX-1; i>0; i = i-1){
        j = rand() % (i+1);
        t = a[j]; a[j] = a[i]; a[i] = t;
    }
}

void mostrar_color(const int a[]){
  int i;
  for(i=0; i<MAX; i=i+1)
    printf("%s%02d%s%s",(i<1||*(i+a)>=*(i-1+a))?AZUL:
      MAGENTA,a[i],RESET,(i<(MAX-1))?",":"");
  printf("\n");
}

main(){
    int lista[MAX];
    srand(time(0));
    mezclar(lista);
    mostrar_color(lista);
}
  1. Compilar este programa y ejecutarlo. ¿Qué hacen las funciones mezclar y mostrar_color?

Este ejercicio consiste en implementar una función que haga el ordenamiento del arreglo, siguiendo el algoritmo “Gnome Sort” que vimos hace un par de semanas.

Recordamos el algoritmo. Un duende del jardín ordena una fila de macetas de distintos tamaños, desde la más pequeña hasta la más grande. La única acción que puede hacer el duende (aparte moverse) es intercambiar dos macetas una al lado de la otra.

El duende empieza por el principio del arreglo (posición 0) y termina cuando llega al final:

  • si está mirando al primer elemento de la lista (a[0]), o si el elemento que mira es mayor o igual al elemento que está justo a la izquierda (a[i] >= a[i-1]), entonces hace un paso a la derecha sin intercambiar nada.
  • si el elemento que está mirando es menor al que está justo a la izquierda (a[i] < a[i-1]), entonces intercambia los dos valores, y hace un paso a la izquierda.

El duende repite este proceso hasta que llegue al final del arreglo.

  1. Agregar al programa una función void gnome_sort(int a[]), que consiste en ordenar el arreglo dado como parámetro, siguiendo el algoritmo Gnome Sort (ordenamiento del duende).

  2. Hagamos más visual y dramático a este programa. Dentro de la función gnome_sort, agregar una llamada a mostrar_color después de cada intercambio de valores en el arreglo.

    Probar aumentando el valor de la constante MAX.

    Usar la función usleep en el bucle de la función gnome_sort para hacerlo más lento.

  3. Hay muchos más algoritmos para ordenar arreglos. Se distinguen por varias características:
    • algunos operan sin usar memoria externa (son in situ), mientras otros usan más memoria
    • algunos tienen un mejor rendimiento que otros en velocidad
    • algunos son más faciles de entender que otros

    El algoritmo del duente es in situ y es fácil de entender. Otro que vamos a implementar ahora es selection sort.

    En este video se puede ver un ejemplo de selection sort con naipes (no hace falta sonido):

    Sea a el arreglo de entrada, de tamaño MAX. Una descripción del algoritmo sería:

    • buscar el mínimo elemento de la lista; sea k su índice
    • intercambiar el elemento de índice k con el elemento de índice 0
    • buscar el mínimo elemento en el resto de la lista (de la posición 1 a la posición MAX − 1); sea k su índice
    • intercambiar el elemento de índice k con el elemento de índice 1
    • buscar el mínimo elemento en el resto de la lista (de la posición 2 a la posición MAX − 1); sea k su índice
    • intercambiar el elemento de índice k con el elemento de índice 2
    • etc. repetir hasta el final de la lista

    A medida que progresa el proceso, el tramo inicial de la lista se vuelve ordenado. Al final del proceso, toda la lista se encuentra ordenada.

    Para implementar este algoritmo necesitamos dos índices:

    • i: indica la próxima posición que tiene que recibir el mínimo del resto de la lista
    • j: usamos para recorrer el resto (desordenado) de la lista para buscar el mínimo.

    Se tratan de dos bucles anidados, usando respectivamente el indice i y j. Dentro de este segundo bucle, usamos una variable k para guardar el índice del elemento mínimo.

    En ordenar.c, implementar el algoritmo en una función void selection_sort(int a[]) y usarlo en el programa que tenemos.

    Imprimir la lista después del sort para asegurarse que queda ordenada.

    Dentro de la función selection_sort imprimir la lista después de cada intercambio de valores.
